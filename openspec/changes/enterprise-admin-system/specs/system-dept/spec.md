## ADDED Requirements

### Requirement: 部门 CRUD
系统 SHALL 提供部门的增删改查功能，部门以树形结构组织。

#### Scenario: 新增部门
- **WHEN** 管理员填写部门信息（部门名称、上级部门、负责人、排序、状态）并提交
- **THEN** 系统创建部门记录

#### Scenario: 修改部门
- **WHEN** 管理员修改部门信息并提交
- **THEN** 系统更新部门记录

#### Scenario: 删除部门
- **WHEN** 管理员删除部门
- **THEN** 系统检查该部门是否有子部门或关联用户，有则拒绝删除

#### Scenario: 查询部门树
- **WHEN** 管理员访问部门管理页面
- **THEN** 系统返回树形结构的部门列表

### Requirement: 部门数据权限隔离
系统 SHALL 支持按部门隔离数据，根据角色的数据权限范围过滤查询结果。

#### Scenario: 本部门及以下
- **WHEN** 角色数据权限为"本部门及以下"
- **THEN** 查询结果包含本部门及其所有子部门的数据
